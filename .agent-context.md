# Agent Context: ISPPOS

Concise entry point for AI agents working in this repo.

## Current App Shape
- Expo React Native app. Entry: `App.js` -> `src/navigation/AppNavigator.js`.
- Active app navigator is `AppNavigator.js`; legacy wrappers/navigator were moved under `old/`.
- SQLite-first/offline-first: local SQLite is the operational source for app behavior. Supabase is used for license lookup/auth data and sync transport.
- Startup gating is strict: `DB_READY` and `INITIAL_SYNC_READY` must be true before opening data screens.
- Initial sync progress is real (table-based), not timer-based; progress messages are shown in Arabic via `LoadingOverlay`.
- Business logic belongs in `src/services/`; screens should call services and render UI.
- Use `src/services/database.js` barrel exports from screens unless a service has a specific reason to import another service directly.

## Hard Rules
- Minimal safe diffs. Do not refactor unrelated code.
- Do not bypass SQLite for business writes.
- Any mutation should update SQLite, enqueue `sync_queue`, notify local subscribers, and be sync-safe.
- Supabase payloads must be schema-safe per-table (strip local-only fields like `synced`, `is_deleted`, `deleted_at`, `delete_reason`, pending flags).
- Respect `project_id`, `phase_id`, soft-delete fields, and closed-phase read-only behavior.
- If usage is uncertain, leave code in place and document uncertainty.
